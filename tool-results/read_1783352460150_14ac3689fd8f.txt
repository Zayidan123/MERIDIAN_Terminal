    30тЖТ## Tentang Proyek
    31тЖТ
    32тЖТ**MERIDIAN Terminal** adalah platform riset dan trading multi-aset yang menyatukan empat fungsi dalam satu sistem: riset/dashboard, sinyal & alert, manajemen risiko/portofolio, dan eksekusi otomatis (paper + live). Dibangun mengikuti PRD yang mensyaratkan standar tampilan dan keandalan data setara institusi finansial profesional.
    33тЖТ
    34тЖТ**Prinsip inti yang tidak bisa ditawar:** seluruh data yang ditampilkan harus data pasar nyata тАФ tidak ada data simulasi, dummy, atau placeholder dalam bentuk apa pun, di lingkungan produksi maupun development. Ketika sumber data down atau rate-limited, sistem menampilkan status "Source unavailable" secara eksplisit, bukan angka karangan.
    35тЖТ
    36тЖТ### Mengapa dibangun?
    37тЖТ
    38тЖТTrader/investor yang mengelola lebih dari satu kelas aset biasanya harus berpindah antar banyak tool terpisah (exchange app, broker saham, TradingView, spreadsheet risk management manual). Ini memecah fokus dan memperlambat pengambilan keputusan. MERIDIAN menyatukan riset, sinyal, dan manajemen risiko lintas aset dalam satu tempat.
    39тЖТ
    40тЖТ### Untuk siapa?
    41тЖТ
    42тЖТSingle-user (pemilik platform), berperan sebagai trader/investor aktif lintas aset. Use case inti: cek kondisi market pagi hari lintas 3 kelas aset dalam satu layar, terima alert saat kondisi tertentu terpenuhi, evaluasi risiko portofolio sebelum entry baru.
    43тЖТ

    46тЖТ## Tujuan Akhir
    47тЖТ
    48тЖТTujuan akhir proyek ini adalah menjadi **terminal trading pribadi yang lengkap dan self-hosted** dengan kemampuan:
    49тЖТ
    50тЖТ1. **Riset terpadu** тАФ satu dashboard untuk crypto, saham IDX, forex & gold dengan chart teknikal + fundamental.
    51тЖТ2. **Sinyal otomatis** тАФ deteksi anomali volume, breakout, RSI ekstrem, return anomaly, dan drawdown breach dari data historis nyata.
    52тЖТ3. **Manajemen risiko institusional** тАФ exposure tracking, position sizing, VaR, drawdown monitor, correlation matrix.
    53тЖТ4. **Eksekusi otomatis** тАФ paper trading (data live, eksekusi simulasi) sebagai default, dengan jalur menuju live execution untuk crypto via CCXT.
    54тЖТ5. **Integritas data 100%** тАФ setiap angka yang tampil bisa dilacak ke sumber pasar nyata + timestamp sinkron.
    55тЖТ6. **Local-first** тАФ berjalan penuh di hardware kelas menengah tanpa bergantung mutlak pada cloud berbayar.
    56тЖТ
    57тЖТ**Bukan tujuan:** menjamin profit. Seluruh sinyal bersifat probabilistik dan alat bantu keputusan, bukan nasihat keuangan.

    61тЖТ## Fitur yang Sudah Dikerjakan
    62тЖТ
    63тЖТ### тЬЕ Fase 0 тАФ Data Layer
    64тЖТ- Pipeline data nyata untuk 3 kelas aset (crypto via Binance, IDX/forex/gold via Yahoo Finance).
    65тЖТ- Normalisasi ke skema internal seragam (OHLCV).
    66тЖТ- Setiap request ke data source dicatat ke `data_source_health_log` (status, latency, error).
    67тЖТ- Retry & backoff untuk API yang gagal, tanpa fallback ke data karangan.
    68тЖТ- **Persistensi `price_ohlcv`** тАФ candle disimpan ke DB lokal saat di-fetch, siap untuk backtesting.
    69тЖТ
    70тЖТ### тЬЕ Fase 1 тАФ Dashboard Riset
    71тЖТ- Watchlist lintas aset (19 instrumen default: 7 crypto, 7 IDX, 4 forex, 1 gold).
    72тЖТ- Chart candlestick custom (SVG) dengan MA20 overlay, volume bars, last-price tag.
    73тЖТ- Indikator teknikal: MA20, MA50, EMA12, RSI14, MACD, volume z-score.
    74тЖТ- Panel fundamental saham: PER, PBV, ROE, EPS, Revenue, Graham Number (dari Yahoo Finance).
    75тЖТ- Panel fundamental crypto: Market Cap, FDV, Circulating/Total/Max Supply (dari CoinGecko).
    76тЖТ- **Provenance Bar** di setiap panel тАФ source + waktu sync terakhir (signature element PRD ┬з13).
    77тЖТ
    78тЖТ### тЬЕ Fase 2 тАФ Signal & Alert Engine
    79тЖТ- Custom alert berbasis threshold: price, %change 24h, volume spike (z-score), RSI, price vs MA.
    80тЖТ- Engine evaluasi alert terhadap data real (polling 60s) тЖТ trigger SignalEvent.
    81тЖТ- Scanner anomali (polling 180s): VOLUME_SPIKE, BREAKOUT, RSI_OB, RSI_OS, ANOMALY (return anomaly vs baseline statistik).
    82тЖТ- Baseline statistik dihitung dari data historis nyata (mean/std log-returns), bukan threshold arbitrer.
    83тЖТ- Dedup 1 jam per (instrument, signalType) untuk hindari spam.
    84тЖТ- **Notifikasi Telegram Bot** тАФ setiap WARN/CRITICAL signal + alert trigger dikirim ke chat Telegram (fire-and-forget, never throws).
    85тЖТ
    86тЖТ### тЬЕ Fase 3 тАФ Risk & Portfolio Management
    87тЖТ- Tracking posisi aktif lintas aset (manual entry, live mark-to-market).
    88тЖТ- Kalkulasi exposure per aset & per kelas aset (% dari total portofolio).
    89тЖТ- Position sizing calculator berbasis % risiko per trade vs equity.
    90тЖТ- Drawdown monitor (current vs historical max) dari reconstructed equity curve.
    91тЖТ- **Drawdown alert trigger (FR-3.4)** тАФ SignalEvent DRAWDOWN_BREACH saat drawdown melewati -10% (WARN) / -20% (CRITICAL), + Telegram notification.
    92тЖТ- Correlation matrix antar aset (Pearson, pairwise, daily log-returns 3 bulan).
    93тЖТ- VaR (Value at Risk) parametrik 95% dari portfolio return series.
    94тЖТ
    95тЖТ### тЬЕ Fase 4 тАФ Execution Bot
    96тЖТ- **Paper trading mode** тАФ eksekusi disimulasikan, tapi harga fill = harga live real-time dari Binance.
    97тЖТ- **Live execution crypto** via CCXT (Binance) тАФ siap, tapi default off (PAPER).
    98тЖТ- **Kill-switch** manual (tombol darurat) + otomatis (auto-fire saat daily drawdown > threshold).
    99тЖТ- **Hard caps** di level kode: max per-order USD, max per-day USD (configurable).
   100тЖТ- **Anomaly detection** тАФ jika >5 order dalam 60s тЖТ auto kill-switch + alert.
   101тЖТ- **Large-order confirm** тАФ order >80% per-order cap butuh konfirmasi manual.
   102тЖТ- **Audit log tamper-evident** тАФ hash chain (setiap row = sha256(prevHash + content)), verifiable.
   103тЖТ- **Process isolation** тАФ execution bot berjalan sebagai mini-service terpisah (port 3002), bukan di proses Next.js.
   104тЖТ- **MT5 (forex/gold live) DEFERRED** тАФ Python-only API, tidak bisa diimplementasikan di stack TypeScript.
   105тЖТ
   106тЖТ### тЬЕ Security & Infrastructure
   107тЖТ- **Autentikasi** NextAuth v4 + bcrypt (password hash) + optional 2FA/TOTP (otplib).
   108тЖТ- **CSP, CSRF, rate-limit** via Next.js middleware + next.config headers.
   109тЖТ- **WebSocket real-time** тАФ mini-service socket.io (port 3001) broadcast harga live dari Binance WS + Yahoo polling, dengan flash animation + LIVE indicator di UI.
   110тЖТ- Dark mode institusional (palette `#0B0E13`/`#151920`/`#262B33`, aksen gain `#2E9E6D` / loss `#C7484B` / brand `#3B5FE0`).
   111тЖТ- Font: IBM Plex Sans (heading), Inter (body), JetBrains Mono (semua angka finansial).
   112тЖТ- Responsive (mobile-first), sticky footer, nav rail persisten.

   386тЖТ## Keamanan
   387тЖТ
   388тЖТImplementasi keamanan mengikuti PRD ┬з16 (Threat Model тЖТ Secure Coding Guardrail). Yang sudah:
   389тЖТ
   390тЖТ- тЬЕ **Secrets via env** тАФ tidak ada API key hardcoded di kode (`.env` di-gitignore).
   391тЖТ- тЬЕ **Password bcrypt-hashed** тАФ `MERIDIAN_ADMIN_PASSWORD_HASH` atau auto-hash dari plaintext env.
   392тЖТ- тЬЕ **2FA/TOTP** тАФ optional via `MERIDIAN_TOTP_SECRET` (otplib).
   393тЖТ- тЬЕ **Session cookie** тАФ httpOnly, SameSite=Lax, 8h expiry.
   394тЖТ- тЬЕ **CSP strict** тАФ `default-src 'self'`, `frame-ancestors 'none'`, `connect-src` terbatas.
   395тЖТ- тЬЕ **CSRF protection** тАФ Origin header check pada POST/PUT/PATCH/DELETE.
   396тЖТ- тЬЕ **Rate limiting** тАФ 600 req/min per IP di middleware.
   397тЖТ- тЬЕ **SQL injection prevention** тАФ Prisma ORM (parameterized), raw SQL di mini-service juga parameterized.
   398тЖТ- тЬЕ **No eval/exec** тАФ tidak ada `eval()` di kode.
   399тЖТ- тЬЕ **Process isolation** тАФ execution bot terpisah dari web app (┬з16.7).
   400тЖТ- тЬЕ **API key trade-only** тАФ dokumentasi + checklist dialog sebelum switch to LIVE.
   401тЖТ- тЬЕ **Hard caps** тАФ max per-order & per-day di level kode (bukan strategi).
   402тЖТ- тЬЕ **Kill-switch** тАФ manual + automatic (drawdown threshold).
   403тЖТ- тЬЕ **Audit log tamper-evident** тАФ hash chain untuk bot actions.
   404тЖТ- тЬЕ **Anomaly detection** тАФ >5 order/60s тЖТ auto kill-switch.
   405тЖТ- тЬЕ **Large-order confirm** тАФ >80% per-order cap butuh konfirmasi.
   406тЖТ
   407тЖТ### тЪая╕П Catatan Keamanan (jujur)
   408тЖТ
   409тЖТ- **Fallback default password** `Meridian@2025` ada di `src/lib/auth.config.ts` sebagai fallback ketika env tidak set. **Wajib diubah** via `.env` sebelum production. Ini deviasi minor dari PRD ┬з16.2 (no secret in code) demi UX "fresh dev box can log in" тАФ trade-off yang disengaja, didokumentasikan di sini.
   410тЖТ- **Bot API token** (`BOT_API_TOKEN`) optional тАФ jika kosong, mini-service execution bot terbuka di local network. Cocok untuk local-only, tapi **wajib set** sebelum expose ke jaringan manapun.
   411тЖТ- **TLS/HTTPS** belum dikonfigurasi di sandbox (HTTP polos). Wajib reverse proxy dengan TLS (Caddy/Nginx) + Tailscale/WireGuard sebelum akses remote (┬з16.4).
   412тЖТ- **Dependency scanning** (`npm audit`) belum di-automate di CI.
   413тЖТ- **Backup** belum ada automation.

   417тЖТ## Tantangan Selama Development
   418тЖТ
   419тЖТ### 1. Rate limit API gratis yang agresif
   420тЖТYahoo Finance `query2` sering return 429 (Too Many Requests) di sandbox. CoinGecko juga. Binance `api.binance.com` sempat IP-banned permanen sehingga harus pindah ke `data-api.binance.vision`. **Solusi:** cache agresif (in-memory TTL), retry+backoff, dan honest error surface ("Source unavailable") sesuai PRD ┬з6 тАФ tidak pernah fabricate data.
   421тЖТ
   422тЖТ### 2. Dev server tidak stabil di sandbox
   423тЖТNext.js dev server berkali-kali mati karena `ECONNRESET` dari koneksi Yahoo yang di-reset, plus masalah orphan process saat background. **Solusi:** pola `setsid` + subshell `(nohup ... &)` untuk daemon survival, restart otomatis.
   424тЖТ
   425тЖТ### 3. Prisma client drift setelah schema push
   426тЖТSetelah `bun run db:push` menambah field baru, Next.js HMR re-compile route handlers tapi tidak invalidate `@prisma/client` module-level cache тЖТ `PrismaClientValidationError: Unknown argument`. **Solusi:** restart dev server setelah setiap schema change (didokumentasikan di worklog).
   427тЖТ
   428тЖТ### 4. Mini-service daemonization
   429тЖТ`nohup bun --hot index.ts &` tidak cukup тАФ process mati ~10s setelah shell exit. `--hot` watcher parent orphan child. **Solusi:** `bun index.ts` (tanpa --hot) + `setsid` + subshell untuk reparent ke PID 1.
   430тЖТ
   431тЖТ### 5. Browser tidak bisa reach localhost:3000 langsung
   432тЖТAgent Browser di sandbox tidak resolve `localhost` dengan andal. **Solusi:** akses via gateway Caddy port 81 yang forward ke 3000.
   433тЖТ
   434тЖТ### 6. Rate limit middleware terlalu ketat
   435тЖТAwalnya 120 req/min тАФ terlalu rendah untuk app data-heavy (quotes 30s, risk 60s, signals 60s, bot status 10s polling = burst). **Solusi:** naikkan ke 600 req/min, masih cukup untuk block brute-force.
   436тЖТ
   437тЖТ### 7. otplib v13 breaking change
   438тЖТv13.4.1 drop v12 `authenticator` singleton. **Solusi:** switch ke `new OTP({ strategy:'totp' })` dengan `await otp.verify({ token, secret, epochTolerance:30 })`.
   439тЖТ
   440тЖТ### 8. Next.js 16 middleware deprecation
   441тЖТ`middleware.ts` deprecated тЖТ `proxy.ts`. Masih bekerja tapi warning. Dibiarkan sebagai `middleware.ts` per spec.
   442тЖТ
   443тЖТ### 9. Yahoo fundamentals (quoteSummary) unreliable
   444тЖТEndpoint `quoteSummary` sering 429/blocked. **Solusi:** defensive parsing, honest null fields, fallback ke "Fundamentals unavailable" UI state.
   445тЖТ
   446тЖТ### 10. CORS + WebSocket routing
   447тЖТFrontend harus connect ke WS service via gateway (`?XTransformPort=3001`), bukan langsung `localhost:3001`. **Solusi:** socket.io-client config dengan `query: { XTransformPort: '3001' }` ke same-origin.
   448тЖТ

   450тЖТ
   451тЖТ## Hal Paling Sulit
   452тЖТ
   453тЖТ### #1: Menjaga integritas data 100% saat sumber tidak stabil
   454тЖТ
   455тЖТIni tantangan filosofis sekaligus teknis. PRD ┬з6 sangat eksplisit: "Dilarang keras menggunakan data simulasi... termasuk saat development atau demo." Tapi Yahoo Finance rate-limit terus-menerus, Binance IP-banned, CoinGecko 429. Godaan untuk "return something, anything" agar UI tidak kosong sangat besar.
   456тЖТ
   457тЖТ**Bagaimana diselesaikan:** Disiplin arsitektural тАФ setiap data-source call return `DataResult<T>` dengan `ok: boolean`. Jika gagal, return `{ ok: false, error }`. API route pakai `fromResult()` yang otomatis return 502 + error message. Frontend render "Source unavailable" badge per-row. Tidak ada satu pun fallback ke data karangan di seluruh codebase.
   458тЖТ
   459тЖТ### #2: Execution bot dengan audit log tamper-evident
   460тЖТ
   461тЖТMembangun hash chain yang benar (setiap row = sha256(prevHash + canonical content)) di SQLite dengan raw SQL, sambil menjaga tidak break data path, plus anomaly detection + hard caps + kill-switch + process isolation тАФ ini paling kompleks. Subagent butuh context besar dan sempat timeout, harus di-resume.
   462тЖТ
   463тЖТ### #3: Reconciliasi real-time (WebSocket) dengan polling (REST)
   464тЖТ
   465тЖТWebSocket memberikan price ticks, tapi 24h stats (change%, high, low, volume) butuh REST polling. Frontend harus overlay live price di atas polled quote tanpa conflict, plus flash animation yang re-fire pada back-to-back ticks (butuh `key={live-${time}}` untuk force remount). Detail kecil tapi tricky.
   466тЖТ
   467тЖТ### #4: Dashboard institusional yang dense tanpa library chart berat
   468тЖТ
   469тЖТPRD ┬з13 minta "density over decoration, presisi, bukan tampilan konsumen." TradingView Lightweight Charts bagus tapi add dep. **Solusi:** custom SVG candlestick chart (160 baris) dengan grid, volume bars, MA overlay, last-price tag тАФ full control, zero dep, institutional look.
