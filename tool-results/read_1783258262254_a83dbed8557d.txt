   801→   801→Notes for the frontend agent:
   802→   802→- Polling strategy:
   803→   803→  • GET /api/v1/portfolio every 15–30s for the positions table — fast
   804→   804→    (concurrent quotes, no candles).
   805→   805→  • GET /api/v1/risk/summary every 30–60s — heavier (concurrent quotes
   806→   806→    + concurrent 3m candle fetches per position). On a 3-position crypto
   807→   807→    portfolio the end-to-end latency observed was ~1.3s; expect more with
   808→   808→    Yahoo instruments (HTTP 429 retries add ~6s per Yahoo instrument).
   809→   809→  • POST /api/v1/risk/position-size is on-demand (form submit).
   810→   810→- Integrity handling (CRITICAL):
   811→   811→  • A position row in GET /portfolio whose live quote failed will have
   812→   812→    lastPrice/marketValue/unrealizedPnl/unrealizedPnlPct all === null
   813→   813→    AND an `error` string field present (e.g. "HTTP 429"). Render that
   814→   814→    row with a "Source unavailable" badge and dim the PnL column — do
   815→   815→    NOT treat null as 0 or hide the row.
   816→   816→  • In GET /risk/summary, the same failed-quote position contributes 0
   817→   817→    (not null) to perPosition.marketValue/weightPct/pnlPct AND its cost
   818→   818→    basis still counts toward totalEquity. So a portfolio with a large
   819→   819→    failed-quote position will show a low exposurePct — that's correct
   820→   820→    and honest. Cross-reference /portfolio for per-row error display.
   821→   821→- Yahoo instruments (EQUITY/FOREX/COMMODITY) will frequently surface
   822→   822→  HTTP 429 in the per-position `error` field of /portfolio AND be
   823→   823→  excluded from the correlation matrix in /risk/summary (since they
   824→   824→  have no return series). Render them with a "Source unavailable"
   825→   825→  badge without hiding the successful crypto rows.
   826→   826→- VaR is null when fewer than 2 positions have a usable return series
   827→   827→  (e.g. only 1 crypto position) OR insufficient data points. Show
   828→   828→  "Insufficient data" rather than 0 in that case.
   829→   829→- Drawdown is returned as a percentage (negative or zero). The frontend
   830→   830→  should compare currentDrawdown against a user-configurable threshold
   831→   831→  (e.g. -10%) for the alert badge per FR-3.4 (alert UI logic is on the
   832→   832→  frontend in this phase; no server-side alert is raised).
   833→   833→- Correlation matrix is a flat list (not a 2D array). Self-pairs
   834→   834→  (a==a) are included with value 1. Cross-pairs may have value=null
   835→   835→  when overlap is <5 points — render as "—" in the matrix cell.
   836→   836→- RiskSnapshot rows are persisted on every /risk/summary call (fire-
   837→   837→  and-forget). The frontend can later add a GET /risk/snapshots route
   838→   838→  to chart the equity/drawdown/VaR time series — that route is not in
   839→   839→  this task's scope but the data is being collected.
   840→   840→- All epoch times are returned in milliseconds.
